**Стек:** Next.js + Supabase + Tailwind

## Как использовать

1. Скопируйте `.env.example` → `.env.local`
2. `npm run db:push` — применить миграции Supabase
3. `npm run dev` — локальный запуск
4. Зарегистрируйтесь на `/ru/register` или `/en/register`

## Как тестировать

1. Регистрация → редирект на дашборд
2. `/finances` — транзакция + бюджет по категории отображаются в фунтах стерлингов
3. `/finances` — добавить кредитную карту, указать лимит/задолженность/платёж, оплатить её через новую транзакцию и проверить уменьшение долга
4. `/notes` — заметка с markdown, поиск, pin
5. `/tasks` — план + задача, kanban/list, смена статуса
6. `/bookmarks` — ссылка с auto-metadata
7. `/documents` — загрузить документ, подписать, скачать, удалить
8. Колокольчик на дашборде — открыть недельный отчёт за последние 7 дней включая сегодня и отметить его прочитанным
9. `/ai-assessment` — нажать "Получить финансовую оценку" и проверить markdown-отчёт
10. `/profile` — имя, bio, аватар
11. `/en/settings` — переключить язык на English и проверить перевод интерфейса
12. `/[lang]/settings` — в блоке финансов начать новый финансовый месяц: указать дату зарплаты и сумму, проверить появление дохода и перенос лимитов в `/finances`
13. `/[lang]/settings` — ввести `RESET` и очистить финансовый отчёт, проверить отсутствие транзакций/бюджетов/кредиток/финансовых уведомлений
14. Мобильная проверка: открыть DevTools viewport 320/375/430px, проверить отсутствие горизонтальной прокрутки, доступность нижнего меню, колокольчика, фильтров, форм и карточек
15. Logout → `/[lang]/login`

## Ограничения

- Нет импорта банковских выписок
- ИИ-оценка зависит от полноты внесённых данных и не является юридической, налоговой или инвестиционной консультацией
- Command palette (⌘K) — v1.1
- Документы ограничены 50 МБ на файл и разрешёнными mime-типами Supabase Storage
- Нижняя мобильная навигация прокручивается горизонтально, если ширины экрана недостаточно для всех разделов
- Reset финансового отчёта сохраняет категории расходов, чтобы формы добавления транзакций и бюджетов не остались пустыми
- Новый финансовый месяц не удаляет историю: старые транзакции остаются доступными через фильтр месяца, а текущий dashboard считает период от выбранной даты старта
