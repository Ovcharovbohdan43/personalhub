# Недельные Финансовые Отчёты

## Версия / Дата Обновления

- [2026-06-08] – Исправлено: отчёты создаются один раз на календарную неделю и используют выбранный язык интерфейса.

## Назначение

Недельный отчёт показывает пользователю краткую финансовую сводку в колокольчике уведомлений: доходы, расходы, баланс недели, платежи по кредиткам и топ категорий расходов.

## Детальное Описание

- Период отчёта стабилен: календарная неделя с понедельника по воскресенье.
- Уникальность контролируется через `app_notifications` по `user_id`, `type`, `period_start`, `period_end`.
- При открытии дашборда приложение вызывает `getNotifications(locale)`, проверяет наличие отчёта за текущую неделю и создаёт или обновляет один отчёт.
- Если ранее были созданы rolling-дубли за пересекающийся период текущей недели, они удаляются автоматически.
- Текст отчёта создаётся на языке текущего интерфейса: `ru` или `en`.

## Как Использовать

1. Пользователь открывает `/ru` или `/en`.
2. Система получает выбранную локаль из маршрута и передаёт её в генератор уведомлений.
3. В колокольчике появляется один weekly report за текущую календарную неделю.

## Примеры

- `/ru` создаёт заголовок `Недельный отчёт: YYYY-MM-DD — YYYY-MM-DD`.
- `/en` создаёт заголовок `Weekly report: YYYY-MM-DD — YYYY-MM-DD`.

## Как Тестировать

1. Открыть dashboard и убедиться, что появился один отчёт за текущую неделю.
2. Обновить dashboard несколько раз в тот же день: новый отчёт не должен появляться.
3. Открыть dashboard на следующий день той же недели: новый отчёт не должен появляться.
4. Переключить язык в `/settings` и открыть dashboard: отчёт должен отображаться на выбранном языке.
5. Проверить `npm run build`.

## Ограничения

- Отчёт создаётся при открытии dashboard, отдельного cron-job пока нет.
- Неделя считается по серверной дате выполнения Next.js.
- Старые отчёты за прошлые недели не удаляются автоматически.
